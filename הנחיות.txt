Career Radar – Project Instructions

אני בונה פרויקט Python בשם Career Radar.

המטרה היא לבנות מערכת שמוצאת עבורי משרות חדשות מחברות רלוונטיות בתחום החומרה והשבבים.

המטרה הנוכחית

המטרה הנוכחית היא לייצר כניסה אמיתית למשרות בכל החברות.

כלומר: לא מספיק למצוא קישור שנראה כמו משרה, ולא מספיק לספור Job-like URLs.
המערכת צריכה להגיע בפועל לדפי המשרות האמיתיים, לקרוא את התוכן שלהם, ולהחזיר רק משרות רלוונטיות לפי תחום ומיקום.

היעד המעשי כרגע:

1. לכל חברה ברשימה לוודא שהסורק מצליח להגיע לעמודי המשרות עצמם.
2. לחלץ מכל משרה לפחות:

Company
Title
URL
Matched keyword
Matched location, אם זמין

3. לוודא שהסינון נעשה לפי תוכן המשרה האמיתי ולא רק לפי טקסט הקישור.
4. לטפל בכל חברה שלא מאפשרת כניסה אמיתית לדפי המשרות שלה.
5. רק אחרי שכל החברות המרכזיות נכנסות לדפי משרות אמיתיים, לעבור לשיפור UX או פיצ׳רים עתידיים.

כרגע אין צורך ב:

AI
Dashboard
הגשות אוטומטיות
Telegram
התראות

קודם בונים מערכת יציבה שעובדת ונכנסת באמת למשרות.

הריפו הפעיל:

avishailevi/career-radar

main

נקודת ההמשך הנוכחית:

הפרויקט כבר עבר ל-GitHub, ולכן אין לבקש ממני להעתיק קבצים או להעלות קבצים מחדש אם יש גישה לריפו.

המבנה הנוכחי כולל לפחות את הקבצים המרכזיים האלה:

מצב הקוד הנוכחי:

main.py מריץ סריקה לפי כל החברות או לפי שם חברה שמועבר כארגומנט.
ScannerFactory בוחר סורק לפי platform.
WorkdayScanner קיים, אבל כרגע הוא רק עוטף את scan_company הכללי.
scanner.py הוא עדיין הסורק המרכזי בפועל.
DEBUG=True כרגע, כדי לראות למה חברות לא מחזירות תוצאות.
companies.py מכיל את רשימת החברות הנוכחית, כולל NVIDIA, Apple, Broadcom, Intel, Microsoft, Google, Amazon, Qualcomm, Marvell, Cadence, Synopsys, Mobileye, Nuvoton, Tower Semiconductor, Applied Materials, KLA, Nova, SolarEdge, Rafael, Israel Aerospace Industries, Elbit.
keywords.py מכיל מילות חיפוש לתפקידי חומרה, שבבים, וריפיקציה, Board Design, Embedded, Integration וכו׳.

מה כבר עובד לפי ההקשר הקודם:

הרצה כללית של python main.py עובדת ומחזירה לפחות חלק מהמשרות הרלוונטיות.
נמצאו בעבר תוצאות רלוונטיות מ-NVIDIA ומ-Apple.
המערכת כבר יודעת לסרוק כמה חברות ברצף ולהדפיס Relevant jobs found.
נוסף מנגנון Debug שמציג Page URL, Page title, Total links, Usable links, Job-like URLs, Location matches, Keyword matches, Relevant jobs ודוגמאות קישורים.

בעיות פתוחות לפי ההקשר הקודם:

הבעיה המרכזית כרגע אינה רק “למצוא משרות”, אלא לוודא כניסה אמיתית לדפי המשרות בכל חברה.
כל חברה צריכה להיבדק לפי השאלה:

האם הסורק נכנס לעמוד משרה אמיתי וקורא את התוכן שלו?

Intel:

בעבר Intel לא החזירה משרות רלוונטיות.
הבעיה שנראתה: נמצאו Job-like URLs אבל לא נמצאו Location matches או Keyword matches, כנראה בגלל אופן טעינת/סינון Workday או בגלל שהטקסט הדרוש נמצא בדפי הפרטים ולא בדף הרשימה.
יש URL מעודכן ב-companies.py שמכוון ל-Workday עם location filters בישראל.
השלב הבא הוא לבדוק את Intel עם:

python main.py Intel

ולתקן את מה שמונע כניסה אמיתית לדפי המשרות של Intel וחילוץ תוכן המשרה.

Microsoft:

בעבר Microsoft כן הציגה קישורי משרות, למשל Security Research Intern, Software Engineer - Fabric Platform, E2E Program Manager.
הבעיה שנראתה: היו Location matches אבל Keyword matches נשאר 0, ולכן Relevant jobs היה 0.
הסורק הכללי מוגדר לקרוא דפי פרטים עבור platform=microsoft, ולכן אם עדיין אין התאמות צריך לבדוק האם דפי הפרטים נטענים, האם הטקסט מכיל מילות מפתח, או האם צריך להוסיף מילות מפתח מדויקות יותר.
השלב הבא הוא לבדוק את Microsoft עם:

python main.py Microsoft

ולתקן את מה שמונע כניסה אמיתית לדפי המשרות של Microsoft וחילוץ תוכן המשרה.

עדיפות עבודה כרגע:

1. לבדוק את מצב הריפו וה-branch לפני כל שינוי.
2. לוודא כניסה אמיתית לדפי משרות, לא רק זיהוי קישורים.
3. לתקן קודם את Intel.
4. לתקן אחר כך את Microsoft.
5. לעבור חברה-חברה ולוודא שכל אחת מחזירה משרות אמיתיות או Debug ברור למה לא.
6. להריץ בדיקות נקודתיות:

python main.py Intel
python main.py Microsoft

7. רק לאחר מכן להריץ:

python main.py

8. לא להוסיף פיצ׳רים חדשים לפני שהכניסה לדפי המשרות בכל החברות המרכזיות יציבה.

קרא את הקוד ישירות מהריפו.
אל תניח שהקבצים המקומיים הם הגרסה העדכנית.
אל תבקש ממני להעלות קבצים שכבר קיימים בריפו.

בצע שינויים ישירות בריפו כאשר ביקשתי ממך לשנות קוד או מסמכים.
עדכן רק את הקבצים הנדרשים.
בצע Commit עם הודעה קצרה וברורה.
אם עובדים עם Branch, המשך לעבוד על אותו Branch אלא אם התבקש אחרת.
אם אינך מצליח לגשת לריפו, אמור זאת במפורש והסבר מה חסר במקום לנחש.
