GitHub הוא מקור האמת

הפרויקט מנוהל ב-GitHub.

הריפו הראשי:

avishailevi/career-radar

GitHub הוא ה-Source of Truth של הפרויקט.

כאשר יש חיבור פעיל ל-GitHub:

קרא את הקוד ישירות מהריפו.
אל תניח שהקבצים המקומיים הם הגרסה העדכנית.
אל תבקש ממני להעלות קבצים שכבר קיימים בריפו.
אל תבקש ממני להעתיק קוד מהצ'אט אם ניתן לעדכן אותו ישירות בריפו.
קרא את מצב ה-main, ה-branch וה-PR הרלוונטיים לפני שאתה מציע שינויים.
בצע שינויים ישירות בריפו כאשר ביקשתי ממך לשנות קוד או מסמכים.
עדכן רק את הקבצים הנדרשים.
בצע Commit עם הודעה קצרה וברורה.
אם עובדים עם Branch, המשך לעבוד על אותו Branch אלא אם התבקש אחרת.
אם אינך מצליח לגשת לריפו, אמור זאת במפורש והסבר מה חסר במקום לנחש.

כאשר אני כותב "המשך"

אין להתחיל מחדש.
אין לשכתב את הפרויקט.
אין לבקש ממני שוב קבצים שכבר נמצאים ב-GitHub.

קודם בדוק:

מהו ה-branch הפעיל או הרלוונטי.
מהו ה-commit האחרון.
מהו ה-PR האחרון, אם קיים.
מה מצב הקוד בריפו.

רק לאחר מכן המשך בדיוק מהמקום שבו עצרנו.

לפני כל שינוי

כתוב:

מטרת השינוי

במשפט אחד.

אחרי כל שינוי

דווח בקצרה:

Changed files

רשימת הקבצים שבאמת השתנו בלבד.

Commit

שם/הודעת ה-Commit או SHA אם קיים.

Run

הפקודה שאני צריך להריץ, לדוגמה:

python main.py

Expected Output

תאר בדיוק מה אני אמור לראות.

אם נדרש ממני לבצע פעולה מקומית, למשל git pull, הרצת בדיקה או פתיחת Pull Request, כתוב זאת במפורש.

עבודה עם קוד

כאשר יש גישה ל-GitHub, אין צורך לשלוח לי קבצים מלאים בצ'אט.

במקום זה:

קרא את הקבצים מהריפו.
שנה אותם ישירות ב-GitHub.
בצע Commit.
דווח מה השתנה ואיך לבדוק.

רק אם אין גישה ל-GitHub, או אם ביקשתי במפורש לראות קובץ, שלח קובץ מלא בלבד ולא קטעי קוד.
